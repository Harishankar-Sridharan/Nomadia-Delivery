# 6. understanding the mission parameters

### 1. Understanding the mission parameters <a href="#toc225775642" id="toc225775642"></a>

Mission Parameters are the core data elements that define the characteristics, objectives, and execution details of each mission. Understanding these parameters is essential for ensuring that missions are properly configured, assigned, and executed in the field.

These parameters provide both operational and logistical information used by

coordinators, dispatchers, and deliverers.

### 2. Access Mission Parameters <a href="#toc199075776" id="toc199075776"></a>

From the **Mission page**

Description

Mission

Barcode

Automatically generated by Nomadia Delivery to uniquely identify the mission.

Agency - Mandatory

**Mandatory field**. Specifies the agency responsible for handling the mission. See _Manage Agencies_.

Subcontractor

The third-party delivery service provider contracted to complete the mission.

Sender Barcode

Barcode supplied by the contractor (optional).

Parent Container ID

Identifies the main container or group to which this item belongs. Used in scenarios where several missions are consolidated into one container.

Grouping key

Missions linked to the same customer are grouped to simplify handling for the deliverer.

Container type

Specifies the type of container (box, pallet, crate, etc.) used for the item.

Origin

Indicates the original location from where the delivery is dispatched.

Scheduled deliverer

Name of the person assigned to perform the delivery

Priority

Indicates the urgency or importance of the mission (e.g., High, Medium, Low).

Status configuration type

Defines which status workflow applies to this mission

One type of password

A security code type (e.g., PIN or OTP) that may be required for mission verification

Mission Status

Shows the current progress of the mission (e.g., Waiting, Delivered).

Sub-status

A more granular state within the main mission status (e.g., "Delivered to the neighbour, dropped at the drop step, Delivered to the consignee").

Motive

Explains the reason for the current sub-status (e.g., "No access" or "Customer not available").

Mission type

**Delivery** – Assigns a mission to deliver goods from a depot or agency to the customer’s location.

**Pickup** – Assigns a mission to collect goods or packages from a customer or specific site.

**Chained** **Pickup** **and** **Delivery** – Combines a pickup and delivery within a single mission flow, ensuring continuous handling of goods between two points.

**Visit** – Used for non-logistic missions such as inspections, maintenance, or customer appointments without goods transfer.

**Drop Shipping** – Manages direct shipments from a supplier to the customer, bypassing the depot or intermediary warehouse.

Provisional date

Tentative date assigned for the mission, subject to change.

Main depot scan date

Date and time when the item was scanned at the central warehouse or depot.

Storage place

The specific location or zone within the depot where the item is stored.

Fixed visit duration

Estimated or standard time (in minutes) expected to complete the delivery/pickup at the address.

Compatibility with the resources

Indicates whether the delivery is compatible with the assigned resource (e.g., vehicle size, agent skill).

Require all skills to be compatible

If checked, the resource must match all required skills for the mission (e.g., heavy lifting, ID verification).

Delivery to be executed before any pickup

Enforces the sequence of deliveries before any pickups in a combined mission

Keys

Indicates whether physical keys or codes are required to access delivery location.

Position

Order of the mission in a planned route (e.g., 1st stop, 2nd stop).

Parcel

Product type

Specifies the type/category of product (e.g., electronics, groceries).
